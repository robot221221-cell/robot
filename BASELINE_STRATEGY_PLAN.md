# 双臂协同项目：Baseline 优先策略与优化路线（冻结版）

> 目的：把当前共识固化为可执行策略，避免“边改边测”导致结论失真。

## 1. 当前系统定位

当前已具备可跑通的完整链路：

- `GA` 任务分配与排序
- `Bi-RRT` 几何避障路径规划
- `TOPP-RA` 时间参数化
- 并行调度（冲突预测、等待/回退、防振荡）

结论：系统**可行**，但当前阶段属于 `Baseline` 水平，存在“碰撞检测开销高、规划耗时大、路径质量可提升”的改进空间。

---

## 2. 核心原则（论文严谨性）

1. **先 Baseline，后优化**：未建立对照前禁止引入新优化。
2. **一次只改一个变量**：每轮改动必须可归因。
3. **固定实验条件**：场景、seed、参数、机器环境保持一致。
4. **保留完整证据链**：脚本、配置、日志、汇总、图表必须可追溯。

---

## 3. Baseline 的正式定义

为避免争议，采用双基线定义：

### B0-Algorithm（算法基线）
- 使用当前最初可运行算法链路（GA→Bi-RRT→TOPP-RA→并行调度）
- 不加入新策略（如 Lazy/层级过滤/RRT*/Informed/CCD）

### B0-System（系统基线）
- 在 B0-Algorithm 基础上，保持原始模型和原始碰撞检测流程
- 不做几何简化（如胶囊替代）

后续任何优化结果必须同时对比 B0-Algorithm / B0-System。

---

## 4. 为什么优化顺序必须这样排

当前最主要瓶颈是碰撞检测（`mj_forward` 调用开销）。

若在检测仍慢时直接上 `RRT*`（重布线 + 大量邻域计算），会导致总体时间不可接受。

因此正确顺序是：

1. **底层减负（模型）**：几何简化（如胶囊体）
2. **策略减负（检测）**：Lazy 检测 + 层级筛选
3. **路径寻优**：Bi-RRT*（在可承受算力下追求更优路径）
4. **收敛加速**：Informed Sampling
5. **终极增强（可选）**：CCD（复杂度最高，放最后）

---

## 5. 分阶段执行计划

## 阶段 A：冻结并完成 Baseline（必须先完成）
- 冻结代码版本（commit）
- 冻结参数配置与 seed 集
- 冻结各场景 GA 最优解（easy/medium/hard 的 solution JSON 固定）
- 跑完 easy/medium/hard 多 seed 统计
- 形成 baseline 报告（success、耗时、质量、稳定性）

**通过门槛**：
- 可重复复现实验
- 指标定义稳定
- 输出文件结构固定

## 阶段 B：底层减负（系统优化）
- 模型简化（胶囊等）
- 与原模型做碰撞判定一致性抽检

**通过门槛**：
- 检测速度显著提升
- 漏检率可接受（理想为 0）

## 阶段 C：检测策略减负（算法前置加速）
- Lazy collision checking
- 层级筛选（宽相位→窄相位）

**通过门槛**：
- 检测调用次数明显下降
- 成功率不下降

## 阶段 D：路径寻优
- Bi-RRT*
- 在固定预算下比较路径质量与耗时

## 阶段 E：收敛加速
- Informed Sampling
- 比较同预算收敛效率

## 阶段 F：高阶安全增强（可选）
- CCD（先局部高风险段验证，再考虑全局）

## 6. 统一评估指标（success-first）

主指标（优先级从高到低）：
1. `success`
2. `parallel_makespan_s`
3. `speedup_vs_serial`
4. `deadlock_wait_count`

次指标：
- `wait_time_total_s`
- `finish_gap_s`
- 路径长度/拐点数
- 碰撞检测调用次数与耗时
- `jump_summary`

瓶颈分解指标（必须单列）：
- `planning_time_s`（几何规划耗时）
- `scheduling_time_s`（调度推演耗时）
- `collision_check_time_s`（碰撞检测总耗时）
- `rrt_nodes`、`rrt_iterations` / `rrt_tree_depth`

统计口径：
- 每个场景至少 5~10 seeds（建议 10）
- 统一报告 `Mean ± SD`
- 对照实验必须使用同一 seed 集

## 9. 论文消融闭环映射（写作时直接对应）

为保证“每一步改动都有证据”，建议固定以下对照关系：

1. **阶段 1（模型简化：胶囊体）**
	- 对照：`B0-System` → `Stage-1`
	- 论点：几何降维显著降低碰撞检测开销。

2. **阶段 2（Lazy + 层级筛选）**
	- 对照：`Stage-1` → `Stage-2`
	- 论点：在不牺牲成功率前提下，进一步减少冗余检测调用。

3. **阶段 3（Bi-RRT*）**
	- 对照：`Stage-2` → `Stage-3`
	- 论点：规划时间可能上升，但路径长度/拐点下降，执行 `makespan` 改善。

4. **阶段 4（Informed Sampling）**
	- 对照：`Stage-3` → `Stage-4`
	- 论点：在保持路径质量的同时，降低 RRT* 收敛所需节点数与时间。

5. **阶段 5（可选 CCD）**
	- 对照：`Stage-4` → `Stage-5`
	- 论点：提升高风险段安全边界，作为工程增强项而非主线指标来源。
